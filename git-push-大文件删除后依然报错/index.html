现象描述

一顿git add . git commit 操作行云流水，在git push的时候报错如下：

原来是因为.vscode下存在的大文件，导致无法push。.vscode对项目并不重要，于是打算删掉再push。

本地删除大文件后用git add .添加到暂存区

Push依然报错存在大文件。

本地删除大文件后用git rm更新暂存区

看到同样的大文件报错，我误以为在本地删除后用git add .并没把暂存区里的大文件删除掉，而这个和我的理解是不一致的。为了解决我的疑惑，我回滚到上一个版本（这里是恢复之前本地被删除的大文件，其实想想为啥还能回退就知道为什么删除后依然会报错了），然后重新在本地删除并用git rm来更新暂存区。

  • 回退到上一版本
  • 删除，添加并提交

Push依然报错，我用git log -p查看确认是删掉了，一脸懵逼不知报错里的大文件在哪里。

现象解释

按照报错里给的网址http://git.io/iEPt8g去看了下解决方案，结合一些博客，知道了个大概：作为版本管理工具，git保存了所有的文件历史。所以哪怕删除了一个文件，也只是增加了一条删除的log，但文件依然在`.git`中存在。

所以要彻底删除一个文件，必须从历史中彻底抹去。

以上理解非常浅薄，甚至可能是错的，欢迎大家斧正

解决方案

官方解决

对删除文件，github提供了两种解决方案，详见上述网址：

  1. 针对最近的一次未push的commit进行重写；
  2. 用BFG Repo-Cleaner 或 git filter-branch 重写所有历史。

我的解决

具体看使用场景，对我这样只是想更新远端个人仓库的需求来说，就简单很多：备份修改过的文件（不含大文件），回滚到添加大文件之前的版本，然后将备份的文件添加回来即可。

另外还有个教训：IDE下写代码会产生一些配置文件，非常影响仓库的大小和git操作的速度，甚至会因为大文件而无法push。所以最好不要git add .而是明确写出修改的文件。
